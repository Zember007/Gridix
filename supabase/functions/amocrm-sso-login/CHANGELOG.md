# AmoCRM SSO Login - Changelog

## v2.0.0 - Полная переработка (Текущая версия)

### Изменения в логике

**СТАРАЯ ЛОГИКА (❌ Неправильно):**
- Создавала синтетических пользователей с email вида `{user_id}.{account_id}@amocrm.gridix.internal`
- Автоматически создавала новых пользователей при каждом запросе
- Использовала детерминистичный пароль на основе секрета
- Не проверяла существование связи с AmoCRM в базе данных

**НОВАЯ ЛОГИКА (✅ Правильно):**
1. **Поиск существующего пользователя:**
   - Ищет запись в таблице `crm_connections` по subdomain и crm_type='amocrm'
   - Получает `user_id` из найденной записи
   - Запрашивает email из таблицы `user_profiles`

2. **Обработка ошибок:**
   - Возвращает 404, если подключение не найдено
   - Не создает новых пользователей
   - Требует предварительную авторизацию через OAuth

3. **Генерация токена:**
   - Создает подписанный JWT-подобный токен
   - Включает user_id, email и AmoCRM метаданные
   - Использует HMAC-SHA256 для подписи

### Структура ответа

**Успешный ответ:**
```json
{
  "token": "eyJhbGci...",
  "expires_at": 1709816400,
  "expires_in": 3600,
  "user_id": "uuid-here",
  "email": "user@example.com"
}
```

**Ошибка (пользователь не найден):**
```json
{
  "error": "User not found",
  "message": "No AmoCRM connection found for this account. Please connect your AmoCRM account first."
}
```

### Зависимости

- Таблица `crm_connections` - хранит OAuth подключения по user_id
- Таблица `user_profiles` - хранит email пользователей
- `_shared/sso-token.ts` - утилиты для работы с токенами

### Требования

1. Пользователь должен предварительно авторизоваться через OAuth (функция `amocrm-oauth-callback`)
2. В таблице `crm_connections` должна быть запись с:
   - `crm_type = 'amocrm'`
   - `subdomain` соответствует запросу
   - `user_id` указывает на валидного пользователя

### Безопасность

- ✅ Не создает пользователей без авторизации
- ✅ Проверяет наличие подключения в БД
- ✅ Использует подписанные токены (HMAC-SHA256)
- ✅ Токены имеют срок действия (expires_at)
- ✅ Логирует все важные операции

### Миграция с v1.0.0

Если вы использовали старую версию:
1. Убедитесь, что все пользователи прошли OAuth авторизацию
2. Проверьте наличие записей в `crm_connections`
3. Старые синтетические пользователи больше не используются
4. Обновите клиентский код для обработки ошибки 404
