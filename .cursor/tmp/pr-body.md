## Зачем (контекст)

Нужно привести заголовок и подзаголовок экрана управления агентской сетью к отдельным формулировкам (CRM/реестр партнёров) и не ломать i18n. Дополнительно выровнены строки для сценария дублирования этажей в менеджере квартир для локалей, где не хватало ключей относительно эталона.

## Что сделано

- В `AgencyPartnersHeader` заголовок и подпись переведены на ключи `partners.agencyNetwork` и `partners.agencyNetworkSubtitle`; у подписи добавлен `max-w-[700px]` для читаемости длинного текста.
- Во всех шести языках в `partners.json` добавлены переводы для `agencyNetwork` и `agencyNetworkSubtitle`.
- В `apartmentsManager.json` для локалей ar, he, ka, tr добавлен блок строк для UI дублирования этажа (`floorDuplicate*` и связанные ключи), чтобы паритет с остальными языками.

## Как проверить

1. `pnpm dev:main`, открыть раздел партнёров / агентской сети (экран с `AgencyPartnersHeader`): проверить новый заголовок и подзаголовок на ru и en.
2. При необходимости переключить язык интерфейса на ar/he/ka/tr и убедиться, что нет «сырых» ключей в `partners` и в флоу дублирования этажа в менеджере квартир.
3. Ожидаемый результат: корректные строки без пропусков перевода; вёрстка подписи не растягивается на всю ширину без ограничения.

## Риски / что может сломаться

- Новые длинные подписи на некоторых языках могут переноситься иначе; при узкой колонке стоит визуально проверить ar/he.
- Переводы в ar/he/ka/tr для `apartmentsManager` добавлены пакетом — при ошибках смысла правки только через носителей.
