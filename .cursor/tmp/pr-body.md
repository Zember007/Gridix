## Зачем (контекст)

При скачивании ZIP с медиа проекта внешние ролики (YouTube и др.) раньше не попадали в удобный для офлайн-просмотра вид: хотелось, чтобы в архиве была статическая страница с встраиванием плеера и ссылками, без обращения к нашим API.

## Что сделано

- В ZIP с медиа добавляется отдельный HTML с блоками для внешних видео: где возможно — iframe с официальным embed, опционально превью-картинка и ссылка на оригинал.
- Поддержаны типовые ссылки YouTube (включая shorts/live/embed), Vimeo и Dailymotion; для нераспознанных URL остаётся понятный fallback со ссылкой.
- Локальные файлы по-прежнему собираются в архив; внешние ролики не дублируются как бинарники, а представлены в каталоге через HTML.
- В списке проектов визуально помечены внешние видео (иконка/подсказка), чтобы отличать их от загруженных файлов.

## Как проверить

1. В проекте добавить медиа с внешней ссылкой на YouTube (и при желании Vimeo/Dailymotion).
2. В списке проектов скачать «все медиа» в ZIP.
3. Распаковать архив, открыть добавленный HTML с внешними видео: должны отображаться встраивания или ссылки; локальные файлы открываются из той же папки.
4. Убедиться, что скачивание ZIP по-прежнему завершается без ошибки при смеси локальных файлов и внешних ссылок.

## Риски / что может сломаться

- Политики встраивания (X-Frame-Options / CSP) у отдельных роликов могут блокировать iframe в локально открытом файле — в таких случаях остаётся переход по внешней ссылке.
- Нестандартные или редирект-URL платформ могут не распознаться и попасть только в fallback-ссылку.
- Размер/состав ZIP меняется за счёт дополнительного HTML-файла.
