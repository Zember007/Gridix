# 🧾 Invoice System Module (Выставление счетов)

## 🎯 Цель

Позволить компаниям оплачивать подписку за выбранный пакет или проект через **выставленный PDF-счёт** (Invoice). Система должна быть адаптирована под законодательство Грузии и предоставлять гибкие настройки для Super Admin.

---

## 👤 Роли и права

### 1. Компания (Tenant / Developer)

**Доступные действия:**

- Заполнить реквизиты своей компании (в настройках).
- Выбрать тарифный пакет и метод оплаты.
- Получить PDF-счёт и оплатить его вручную.
- Получать email-уведомления о статусе счёта.

### 2. Super Admin (GRIDIX)

**Доступные действия:**

- Настроить реквизиты платформы GRIDIX для выставления счетов.
- Выбрать язык счётов (грузинский / английский).
- Настроить шаблоны email-уведомлений.
- Просматривать все счета компаний.
- Подтверждать оплату и активировать подписки вручную.

## 💡 Бизнес-логика

1. Компания выбирает тарифный пакет (например, PRO на 3 месяца) для проекта.
2. В методе оплаты выбирает **«Выставить счёт»** (если страна = Грузия).
3. Система формирует **PDF-счёт**:
   - Получатель: GRIDIX (из системных настроек)
   - Плательщик: данные компании
   - Назначение платежа: генерируется автоматически
     ```
     Оплата подписки на 3 месяца по проекту "PRIME" (аккаунт S2 Capital)
     ```
   - Добавляются реквизиты GRIDIX, логотип и печать.
4. PDF-счёт сохраняется в базе (`invoices`) и становится доступен для скачивания.
5. Компания получает **email-уведомление** с темой “Вам выставлен счёт на оплату подписки”.
6. После поступления оплаты Super Admin вручную подтверждает оплату, и система активирует подписку.
7. Компания получает email: “Ваша подписка активирована”.

## 📨 Email-уведомления

### Шаблоны уведомлений (редактируемые Super Admin)

1. **Invoice Created**

   - Тема: “GRIDIX | Вам выставлен счёт”
   - Текст: “Здравствуйте, [CompanyName]. Вам выставлен счёт №[InvoiceNumber] на сумму [Amount] GEL за проект [ProjectName].”
   - Кнопка: “Скачать счёт”

2. **Invoice Paid / Activated**

   - Тема: “GRIDIX | Подписка активирована”
   - Текст: “Подписка по счёту №[InvoiceNumber] успешно активирована. Спасибо за оплату.”

3. **Invoice Pending Reminder**

   - Тема: “GRIDIX | Ожидаем оплату по счёту”
   - Текст: “Ваш счёт №[InvoiceNumber] на сумму [Amount] GEL всё ещё ожидает оплаты.”

## 🌐 Многоязычность

Super Admin может выбрать язык генерации счёта:

- 🇬🇪 Грузинский (основной)
- 🇬🇧 Английский (вторичный)

В зависимости от выбранного языка генерируется шаблон PDF и тексты уведомлений.

## 📄 Структура PDF-счёта

- Логотип GRIDIX

- Дата

### Тело

| Поле       | Значение         |
| ---------- | ---------------- |
| Плательщик | [CompanyName]    |
| Получатель | GRIDIX LLC       |
| Назначение | [PaymentPurpose] |
| Сумма      | [Amount] GEL     |

### Подвал

- Реквизиты GRIDIX (Банк, IBAN, Код)
- Печать
- Подпись “Finance Department GRIDIX”

## 🔄 Сценарий работы

### Сценарий 1: Оплата подписки через счёт (Грузия)

1. Компания **S2 Capital** активирует пакет **PRO** на 3 месяца для проекта **Lemon Garden**.
2. Выбирает способ оплаты → “Выставить счёт (Грузия)”.
3. Система создаёт PDF-счёт:
   ```
   Назначение: Оплата подписки на 3 месяца за проект “Lemon Garden” (аккаунт S2 Capital)
   Сумма: 750 GEL
   ```
4. Компания получает письмо: “Вам выставлен счёт №INV-20251015-023”.
5. Счёт доступен для скачивания из кабинета компании.
6. Компания оплачивает по банковским реквизитам GRIDIX.
7. Финансовый отдел проверяет поступление и подтверждает оплату.
8. Система обновляет статус → **paid** и активирует подписку.
9. Компания получает письмо: “Ваша подписка активирована”.

## 🧠 Технические детали

| Задача               | Технология                                          |
| -------------------- | --------------------------------------------------- |
| PDF генерация        | `pdfmake` или `react-pdf`                           |
| Хранение файлов      | Supabase / Firebase Storage                         |
| Email уведомления    | SMTP или Supabase Trigger                           |
| Админ-панель         | Next.js + Shadcn UI                                 |

## 🔔 Важные детали

- После оплаты **компания не подтверждает** оплату — проверка вручную со стороны Super Admin.
- При необходимости компания может обратиться в **техподдержку в Telegram**.
- Все уведомления гибко настраиваются.

## ✅ Результат

Модуль выставления счётов позволяет безопасно, юридически корректно и удобно работать с компаниями в Грузии. Процесс автоматизирован: компания получает счёт, оплачивает, а платформа управляет статусами и уведомлениями.
