# Управление пользователями - Полное руководство

## Обзор

Компонент `UsersManagement` предоставляет супер-администратору полный контроль над пользователями системы. Все функции теперь полностью рабочие.

## Функционал

### 1. Создание пользователя
- **Кнопка:** "Добавить пользователя"
- **Обязательные поля:** Email, Пароль
- **Опциональные поля:** Полное имя, Компания, Телефон
- **Результат:** Пользователь создается и автоматически подтверждается

### 2. Управление блокировками
- **Блокировка:** Кнопка "Бан" с указанием причины
- **Разблокировка:** Кнопка "Разблокировать" для заблокированных пользователей
- **Статус:** Визуальные бейджи показывают статус пользователя

### 3. Impersonation (Вход под пользователем)
- **Кнопка:** "Войти" для любого пользователя
- **Результат:** Открывается новая вкладка с сессией выбранного пользователя
- **Безопасность:** Супер-админ не теряет свою сессию

### 4. Поиск и фильтрация
- **Поиск:** По email, имени или компании
- **Результат:** Фильтрация в реальном времени

## Безопасность

### Проверка прав доступа
```typescript
const isSuperAdmin = profile?.email === 'inbox@gridix.live';
```

### Аутентификация
- Все запросы проверяют JWT токен
- Edge функция использует Service Role Key
- Проверка прав доступа на уровне функции

## Развертывание

### 1. Развертывание Edge функции
```bash
cd web
supabase functions deploy superadmin-user-management
```

### 2. Настройка переменных окружения
```bash
supabase secrets set SUPABASE_URL=your_url
supabase secrets set SUPABASE_SERVICE_ROLE_KEY=your_key
```

### 3. Проверка работы
- Войдите как супер-админ (`inbox@gridix.live`)
- Перейдите в раздел "Управление пользователями"
- Все функции должны работать

## Структура данных

### Таблица `user_profiles`
```sql
- id (UUID) - ID пользователя
- email (TEXT) - Email пользователя
- full_name (TEXT) - Полное имя
- company_name (TEXT) - Название компании
- phone (TEXT) - Телефон
- created_at (TIMESTAMP) - Дата создания
- updated_at (TIMESTAMP) - Дата обновления
```

### Таблица `banned_users`
```sql
- id (UUID) - ID записи
- user_id (UUID) - ID заблокированного пользователя
- banned_by (UUID) - ID администратора
- reason (TEXT) - Причина блокировки
- banned_at (TIMESTAMP) - Дата блокировки
- unbanned_at (TIMESTAMP) - Дата разблокировки
```

## API Endpoints

### Edge функция: `/functions/v1/superadmin-user-management`

#### POST - Создание пользователя
```json

}
```

## Устранение неполадок

### 1. Функция не развертывается
- Проверьте, что вы в директории `web`
- Убедитесь, что Supabase CLI установлен
- Проверьте права доступа к проекту

### 2. Ошибки аутентификации
- Проверьте, что пользователь имеет email `inbox@gridix.live`
- Убедитесь, что JWT токен валиден
- Проверьте переменные окружения

### 3. Impersonation не работает
- Проверьте, что Edge функция развернута
- Убедитесь, что URL функции правильный
- Проверьте логи в Supabase Dashboard
