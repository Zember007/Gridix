# Widget Updates - Modal & Cache Fixes

## Проблемы и решения

### ✅ Исправлены модальные окна в виджете

**Проблема:** Модальные окна (Dialog, AlertDialog, Sheet и др.) не работали в виджете из-за использования Shadow DOM для изоляции стилей.

**Решение:** 
- Создан специальный контейнер `gridix-portal-container` внутри Shadow DOM
- Обновлены все компоненты с порталами для рендеринга внутри Shadow DOM
- Компоненты автоматически определяют, находятся ли они в контексте виджета

**Обновленные компоненты:**
- `Dialog` 
- `AlertDialog`
- `Sheet`
- `Drawer`
- `Popover`
- `Tooltip`
- `DropdownMenu`

### ✅ Добавлено версионирование для обновления кеша

**Проблема:** При обновлении виджета браузеры использовали закешированные версии CSS и JS файлов.

**Решение:**
- Добавлен timestamp в качестве версии при каждой сборке
- CSS файл автоматически загружается с query параметром `?v={timestamp}`
- При каждом `npm run build:widget` генерируется новая версия

## Как использовать обновленный виджет

### Сборка виджета

```bash
npm run build:widget
```

При каждой сборке:
1. Генерируется уникальная версия (timestamp)
2. Версия выводится в консоль: `Widget build version: 1234567890`
3. CSS файлы загружаются с query параметром для обхода кеша

### Файлы сборки

После сборки в директории `dist-widget/` будут:
- `widget.js` - основной файл виджета
- `style.css` - стили виджета (загружаются с версионированием)

### Пример использования

```html
<!-- Подключение виджета -->
<script src="https://your-domain.com/widget.js"></script>

<script>
  // Инициализация виджета
  GridixWidget.init({
    projectId: 'your-project-id',
    theme: 'light',
    width: '100%',
    height: '600px',
    lang: 'ru'
  });
</script>
```

### Что теперь работает

✅ **Модальные окна** - Dialog, AlertDialog, Sheet и другие компоненты с порталами  
✅ **Popovers и Tooltips** - всплывающие элементы с правильным позиционированием  
✅ **Dropdown меню** - выпадающие списки с правильным рендерингом  
✅ **Автоматическое обновление кеша** - новые версии загружаются без ручной очистки кеша  

## Технические детали

### Shadow DOM Portal Container

Виджет создает специальный контейнер для порталов:

```javascript
// В Shadow DOM создается контейнер
<div id="gridix-portal-container"></div>
```

Все модальные окна и поповеры автоматически рендерятся в этот контейнер, получая доступ к стилям из Shadow DOM.

### Версионирование CSS

Версия внедряется в код виджета на этапе сборки:

```javascript
const widgetVersion = __WIDGET_VERSION__; // "1234567890"
const cssHref = `style.css?v=${widgetVersion}`;
```

При каждой новой сборке генерируется новый timestamp, что заставляет браузеры загружать свежую версию CSS.

## Обновление на продакшене

После сборки виджета:

1. Загрузите новые файлы из `dist-widget/` на сервер
2. Виджет автоматически будет использовать новую версию CSS
3. Пользователи получат обновленную версию без ручной очистки кеша

## Устранение неполадок

### Модальные окна не отображаются

**Проверьте:**
- Виджет инициализирован правильно
- В консоли нет ошибок загрузки CSS
- Shadow DOM создан корректно

**В консоли должны быть сообщения:**
```
GridixWidget: Loading CSS from: ...
GridixWidget: CSS loaded successfully
```

### Старая версия CSS

**Решение:**
- Пересоберите виджет: `npm run build:widget`
- Загрузите новые файлы на сервер
- Проверьте, что CSS загружается с новым query параметром
