# AmoCRM Custom Fields Integration Update

## Изменения

### 1. Создана миграция для таблицы custom fields
- **Файл**: `supabase/migrations/20250915000000-add-amocrm-custom-fields.sql`
- **Описание**: Создана таблица `amocrm_custom_fields` для хранения информации о пользовательских полях AmoCRM
- **Поля**:
  - `project_id` - ID проекта
  - `field_id` - ID поля в AmoCRM
  - `field_name` - Название поля
  - `field_code` - Код поля (если есть)
  - `field_type` - Тип поля
  - `is_required` - Обязательное ли поле
  - `entity_type` - Тип сущности ('leads', 'contacts', 'companies')

### 2. Обновлен OAuth callback
- **Файл**: `supabase/functions/amocrm-oauth-callback/index.ts`
- **Изменения**:
  - Добавлено получение custom fields для контактов и лидов из AmoCRM API
  - Сохранение custom fields в базу данных при авторизации
  - Обновление записей при повторной авторизации

### 3. Обновлена функция создания лидов
- **Файл**: `supabase/functions/create-amocrm-lead/index.ts`
- **Изменения**:
  - Получение custom fields из базы данных вместо API запросов
  - Автоматическое заполнение полей лида данными квартиры (номер, площадь, цена, проект)
  - Улучшенное логирование для отладки
  - Более детальные заметки к лиду

## Преимущества

1. **Производительность**: Не нужно делать дополнительные API запросы к AmoCRM при создании каждого лида
2. **Надёжность**: Custom fields сохраняются в базе данных и доступны даже при проблемах с AmoCRM API
3. **Автоматизация**: Поля квартиры автоматически заполняются в соответствующие custom fields AmoCRM
4. **Гибкость**: Система автоматически адаптируется к структуре полей конкретного аккаунта AmoCRM

## Использование

1. При подключении AmoCRM к проекту система автоматически загрузит и сохранит все custom fields
2. При создании лида система будет использовать сохранённые поля для корректного заполнения данных
3. Если в AmoCRM есть поля для номера квартиры, проекта, площади или цены - они будут автоматически заполнены
