# Интеграция с AmoCRM через OAuth2 (Улучшенная версия)

Данная интеграция позволяет автоматически отправлять заявки с сайта в AmoCRM в виде лидов, используя современный протокол авторизации OAuth2.

## ✨ Новые возможности

- **Настройка в один клик**: Полная настройка интеграции через одну авторизацию
- **Автоматическое получение данных**: Воронки, статусы и пользователи загружаются автоматически из AmoCRM
- **Умный интерфейс**: Выпадающие списки с реальными данными из AmoCRM вместо ручного ввода ID
- **Автоконфигурация**: Основная воронка и первый статус выбираются автоматически
- **Управление токенами**: Автоматическое обновление и обработка истечения токенов
- **Валидация данных**: Встроенная проверка всех настроек

## Настройка AmoCRM

### 1. Создание OAuth2 интеграции в AmoCRM

1. Войдите в свой аккаунт AmoCRM
2. Перейдите в **Настройки** → **Интеграции** → **Создать интеграцию**
3. Выберите тип интеграции: **"Собственная интеграция"**
4. Заполните данные интеграции:
   - Название: "Floorplan Wizard"
   - Описание: "OAuth2 интеграция для получения заявок с сайта"
   - **Redirect URI**: `https://your-domain.com/functions/v1/amocrm-oauth-callback`
   - Права доступа: выберите необходимые права (минимум: создание лидов и контактов)
5. Сохраните интеграцию и скопируйте **Client ID** и **Client Secret**

### 2. Получение ID воронки и статуса

1. Перейдите в **Настройки** → **Воронки и этапы**
2. Найдите нужную воронку и скопируйте её ID из URL или настроек
3. (Опционально) Скопируйте ID нужного этапа в воронке

### 3. Получение ID ответственного пользователя

1. Перейдите в **Настройки** → **Пользователи**
2. Найдите нужного пользователя
3. ID пользователя можно найти в URL профиля пользователя

## Настройка в админ панели

1. Войдите в админ панель
2. Откройте нужный проект для редактирования
3. Перейдите на вкладку **AmoCRM**
4. Заполните следующие поля:
   - **Client ID**: Client ID из созданной интеграции
   - **Client Secret**: Client Secret из созданной интеграции
   - **Поддомен**: часть URL до .amocrm.ru (например, "example")
   - **ID воронки**: ID воронки, куда будут попадать лиды
   - **ID статуса** (опционально): ID конкретного этапа в воронке
   - **ID ответственного** (опционально): ID пользователя, который будет назначен ответственным

5. Нажмите **Сохранить настройки**
6. Нажмите **Авторизовать** для получения токенов доступа
7. В открывшемся окне AmoCRM подтвердите доступ к вашему аккаунту

## Как работает интеграция

1. Пользователь заполняет форму резервации квартиры на сайте
2. Данные отправляются в Edge Function `create-amocrm-lead`
3. Функция:
   - Получает настройки AmoCRM для проекта из базы данных
   - Проверяет валидность токена доступа (обновляет при необходимости)
   - Формирует данные лида с информацией о квартире и контактах
   - Отправляет лид в AmoCRM через OAuth2 API
   - Возвращает результат пользователю

## Структура создаваемого лида

В AmoCRM создается лид со следующими данными:

- **Название**: "Заявка на квартиру {номер_квартиры} - {название_проекта}"
- **Воронка**: указанная в настройках
- **Статус**: указанный в настройках (или первый в воронке)
- **Ответственный**: указанный в настройках
- **Контакт**: создается с именем, email и телефоном клиента

## Troubleshooting

### Ошибка "AmoCRM integration not configured"
- Убедитесь, что настройки AmoCRM сохранены для проекта
- Проверьте, что все обязательные поля заполнены

### Ошибка авторизации в AmoCRM
- Проверьте правильность Client ID и Client Secret
- Убедитесь, что Redirect URI настроен корректно в интеграции AmoCRM
- Убедитесь, что интеграция активна в AmoCRM
- Проверьте права доступа интеграции
- Попробуйте переавторизоваться через кнопку "Переавторизовать"

### Лиды не создаются
- Проверьте ID воронки и статуса
- Убедитесь, что воронка активна
- Проверьте логи в Edge Function

## Безопасность

- Client Secret и токены хранятся в зашифрованном виде в базе данных
- Используется OAuth2 протокол для безопасной авторизации
- Токены автоматически обновляются при истечении срока действия
- Edge Function работает без авторизации, но проверяет существование проекта
- Все запросы логируются для отладки
