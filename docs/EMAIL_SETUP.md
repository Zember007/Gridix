# Настройка Email-уведомлений для приглашений менеджеров

## Обзор

Система автоматически отправляет email-приглашения менеджерам с уникальными ссылками для регистрации. Поддерживается интеграция с сервисом Resend для надежной доставки писем.

## Настройка Resend API

### 1. Регистрация в Resend

1. Зайдите на [resend.com](https://resend.com)
2. Создайте аккаунт
3. Подтвердите домен для отправки писем
4. Получите API ключ

### 2. Настройка переменных окружения

Добавьте следующие переменные в ваш `.env` файл:

```env
# Resend API Configuration
RESEND_API_KEY=re_xxxxxxxxxx
FROM_EMAIL=noreply@yourdomain.com
SITE_URL=https://yourdomain.com
```

### 3. Настройка в Supabase

В панели Supabase перейдите в Settings > Edge Functions > Environment Variables и добавьте:

- `RESEND_API_KEY`: Ваш API ключ от Resend
- `FROM_EMAIL`: Email адрес отправителя (должен быть на подтвержденном домене)
- `SITE_URL`: URL вашего сайта для формирования ссылок

## Структура системы приглашений

### База данных

Система использует следующие таблицы:

- `manager_invitations` - хранит приглашения для незарегистрированных пользователей
- `manager_accounts` - хранит активных менеджеров
- `manager_permissions` - управляет правами доступа менеджеров

### Edge Function

`supabase/functions/send-manager-invitation/index.ts` - обрабатывает отправку email-приглашений

### Компоненты

- `ManagerAccountsManager.tsx` - интерфейс управления менеджерами
- `AcceptInvitationPage.tsx` - страница принятия приглашения

## Процесс приглашения

1. **Создание приглашения**: Застройщик вводит данные менеджера
2. **Проверка пользователя**: Система проверяет, зарегистрирован ли пользователь
3. **Отправка приглашения**: 
   - Если пользователь существует - создается прямая связь
   - Если нет - отправляется email с уникальной ссылкой
4. **Принятие приглашения**: Менеджер переходит по ссылке и создает аккаунт
5. **Активация**: После регистрации менеджер получает доступ к проектам

## Шаблон Email-приглашения

Письмо содержит:

- Персональное приветствие
- Информацию о компании и застройщике
- Список возможностей для менеджера
- Кнопку для принятия приглашения
- Срок действия приглашения (7 дней)

## Безопасность

- Уникальные токены для каждого приглашения
- Ограниченный срок действия (7 дней)
- Проверка статуса приглашения
- Защита от повторного использования

## Fallback режим

Если API ключ Resend не настроен:

- Система создает приглашение в базе данных
- Показывает уведомление о том, что email не отправлен
- Предоставляет кнопки для копирования и открытия ссылки вручную

## Отладка

Для проверки работы системы:

1. Откройте консоль разработчика в браузере
2. Создайте приглашение менеджера
3. Проверьте логи Edge Function в панели Supabase
4. Убедитесь, что письмо пришло на указанный email

## Права менеджеров

Менеджеры получают следующие права:

✅ **Разрешено:**
- Просмотр всех проектов застройщика
- Редактирование проектов
- Создание новых проектов
- Управление квартирами и планировками
- Просмотр настроек

❌ **Запрещено:**
- Удаление проектов
- Изменение данных о компании
- Управление другими менеджерами
- Изменение настроек компании

## Мониторинг

Рекомендуется отслеживать:

- Количество отправленных приглашений
- Процент принятых приглашений
- Ошибки отправки email
- Истекшие приглашения

## Масштабирование

Для больших объемов:

- Настройте лимиты Resend согласно вашему плану
- Рассмотрите использование очередей для отправки
- Добавьте повторные попытки при ошибках
- Настройте мониторинг доставляемости
